# Thin Domain Setup Checklist

Use this checklist for domains that should not be richly filled during setup unless confirmed facts already exist.

## Backlog

Allowed write:

- `docs/backlog/README.md`

Fill only:

- source of truth
- current focus, if existing docs say it clearly
- links to existing todo/doing/done/ideas folders or external tracker
- rules already present in the project

Do not invent backlog items.

## Testing

Allowed write:

- `docs/testing/README.md`

Fill only:

- detected test scripts
- links to existing testing docs
- known gaps already supported by docs or checks

Do not invent a test strategy.

## Decisions

Allowed write:

- `docs/decisions/README.md` or link existing ADR folder

Fill only:

- links to existing accepted decisions

Do not invent decisions during setup.

## Open Questions

Allowed write:

- `docs/open-questions.md`

Fill only:

- real conflicts found during setup
- risky unknowns that could cause wrong work

Do not use this as a backlog.
