# Product Setup Checklist

Use this checklist for the product domain during `$ea-docs setup`.

## Allowed Writes

- `docs/product/README.md`
- `docs/product/vision.md`

## Read Candidates

- root `README.md`
- package or app config names
- existing product, roadmap, feature, scenario, or backlog docs
- current plan or accepted decision notes if provided

Do not read every file. Read only enough to fill the product domain safely.

## Fill

- project purpose
- target users, if supported by docs
- current scope
- explicit non-goals or later scope
- links to feature status or backlog docs

## Fact Labels

Use these labels when useful:

- `Confirmed`: directly supported by current docs or code
- `Inferred`: likely from file names or structure
- `Needs review`: product meaning could be stale or ambiguous
- `Unresolved`: conflicting sources exist

## Rules

- Do not invent product strategy.
- Do not treat old roadmap text as current truth.
- If current scope and old docs conflict, report the conflict.
- Keep product docs concise.
